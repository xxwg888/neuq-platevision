# 明日任务清单：流程理解、PPT、报告

## 1. 先讲懂技术流程

- 主识别路线：整车图 → YOLOv8n-pose 输出检测框、颜色和 4 个角点 → 透视校正 → CRNN-CTC 整牌序列识别 → 输出车牌号。
- 传统对比路线：校正后的车牌裁剪 → 灰度/Otsu 二值化 → 投影法强制字符分割 → 模板匹配或 HOG+KNN 单字符分类。
- 前端展示路线：FastAPI 后端提供 provider，React 前端展示单图识别、批量实验、推理设置和中间过程图。
- 口径提醒：ONNX 模式下的二值化/字符框是辅助展示，不是 CRNN-CTC 的必要输入。

## 2. 报告必须覆盖

- 设计任务和实验条件：蓝牌、绿牌必做，黄牌/白牌作为扩展；准确率、精确率、召回率、误差分析。
- 数据集：CCPD2019、CCPD-Green、CRPD；train/val/test = 11668/2936/2977。
- 模型设计：YOLOv8n-pose 多任务检测与角点定位；CRNN = CNN + BiGRU + CTC。
- 传统方法：模板匹配和 HOG+KNN，作为字符分割和传统识别对照实验。
- 前后端系统：React + Vite 前端，FastAPI 后端，ONNX Runtime CPU 推理，GitHub 开源地址。
- 实验结果：检测 P/R/F1、端到端准确率、字符准确率、颜色分类、分颜色结果、错误案例。
